Design layouts, components and elements in Figma.

Figma is the perfect collaborative software to first define the branding rules, create the design elements and make some tests to give your projects a unity.

Build the design tokens.

Translate every design rule in tokens in Visual Studio Code. Build the elements in html / css using those tokens.

Display on Storybook.

One of the purpose of this design system is to make the life of the developers easy. So they can just copy paste the code or the tokens from storybook.

Collaborative work with Github.

New elements, new design tokens have to be added. Old elements have to be updated. We work as a team and keep this design system updated on the go.

Design system in use.

We use this design system for multiple projects. Thanks to this, we speed up our deployments, create a unity toward those projects.

My role in this project.

My initial role is to define the design rules, layouts and elements of this design system on Figma. But I am also able to use git, dive in the code and make new releases myself.
